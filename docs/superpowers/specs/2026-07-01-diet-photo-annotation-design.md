# 饮食照片营养标注网页设计

## 背景

用户每天拍摄餐食照片，并按固定格式输入食物、单位、热量和蛋白质。系统生成一张带营养信息覆盖层的标注图片，同时保存当天完整的 Markdown 饮食记录。

第一版面向安卓手机 Chrome，部署在 Vercel。应用优先采用纯前端实现，图片和文本都在浏览器本地处理，不上传服务器。

## 目标

- 用户可以用手机相机拍摄或从相册选择一张餐食图片。
- 用户按固定格式输入餐次和食物数据。
- 系统解析输入，计算本餐热量、蛋白质，以及当天累计值。
- 系统生成一张沿用现有样式的 PNG 标注图。
- 用户点击保存后，浏览器下载 PNG 标注图和当天完整 MD 文件到默认下载位置。
- 热量目标和蛋白质目标可编辑，并在浏览器本地记住。
- 当天记录按手机本地日期累计，跨天自动新建记录。
- 用户可以查看当天已保存餐次，并删除错误餐次后重新保存。

## 非目标

- 第一版不自动识别图片中的食物。
- 第一版不自动估算单位、热量或蛋白质。
- 第一版不接入大模型 API。
- 第一版不做账号登录、云端同步或跨设备同步。
- 第一版不要求静默覆盖手机 Downloads 里的同名 MD 文件。
- 第一版不做完整 PWA 离线安装体验。

## 运行环境

- 目标设备：安卓手机。
- 目标浏览器：Chrome。
- 部署平台：Vercel。
- 应用类型：移动端网页。
- 数据处理：浏览器本地 JavaScript。
- 图片生成：HTML Canvas。
- 本地状态：localStorage 或 IndexedDB。
- 文件保存：浏览器下载机制。

## 用户流程

1. 用户打开网页。
2. 页面自动加载上次保存的热量目标和蛋白质目标。
3. 页面按手机本地日期加载当天记录。
4. 用户拍照或选择餐食图片。
5. 用户输入固定格式文本。
6. 用户点击预览。
7. 系统解析文本并显示：
   - 本餐食物表格
   - 本餐热量和蛋白质
   - 今日累计热量和蛋白质
   - 目标热量和蛋白质
8. 系统生成标注图预览。
9. 用户点击保存。
10. 系统把本餐写入当天本地记录。
11. 系统下载 PNG 标注图。
12. 系统下载当天完整 MD 文件。

## 输入格式

固定格式如下：

```text
第一餐
酒酿 / 150g / 120 / 4
香蕉 / 1根 / 105 / 1
鸡蛋 / 2个 / 140 / 12
鱼肉 / 60g / 80 / 12
```

含义：

```text
餐次标题
食物名 / 单位 / 热量kcal / 蛋白质g
```

解析规则：

- 第一行是餐次标题，原样用于标注图顶部标题和 MD 小节标题。
- 从第二行开始，每行表示一个食物条目。
- 每个食物条目必须包含 4 个字段，用 `/` 分隔。
- 食物名和单位字段保留原始文本。
- 热量字段解析为数字，单位固定为 kcal。
- 蛋白质字段解析为数字，单位固定为 g。
- 字段前后的空格自动忽略。
- 空行自动忽略。

输入错误处理：

- 如果第一行为空，提示“请填写餐次标题”。
- 如果某个食物行字段数量不是 4 个，提示具体行号。
- 如果热量或蛋白质不是数字，提示具体行号和字段。
- 出错时不生成保存文件。

## 目标设置

页面提供两个可编辑数值：

- 目标热量，默认 `2700kcal`。
- 目标蛋白质，默认 `135g`。

用户修改后，应用立即保存到浏览器本地。之后再次打开网页时自动恢复。

## 当天累计规则

- 日期以手机本地日期为准，格式为 `YYYYMMDD`。
- 每个自然日独立保存记录。
- 同一天内，每次保存一餐后，该餐加入当天记录。
- 今日累计等于当天所有已保存餐次的热量和蛋白质总和。
- 跨天后自动新建当天记录，不再累计昨天数据。
- 如果用户删除当天某餐，累计值立即按剩余餐次重新计算。

## 本地数据结构

建议按日期保存：

```json
{
  "date": "20260701",
  "targets": {
    "calories": 2700,
    "protein": 135
  },
  "meals": [
    {
      "id": "timestamp-or-uuid",
      "title": "第一餐",
      "createdAt": "2026-07-01T08:30:00+08:00",
      "imageFileName": "IMG_20260701_01.jpg",
      "items": [
        {
          "name": "酒酿",
          "amount": "150g",
          "calories": 120,
          "protein": 4
        }
      ],
      "totals": {
        "calories": 120,
        "protein": 4
      }
    }
  ]
}
```

图片文件本身不需要长期保存进浏览器数据库。保存标注图后，浏览器下载 PNG 文件即可。当天累计只依赖文本结构化数据。

## 标注图模板

沿用当前样式：

- 输出格式：PNG。
- 输出比例：跟随原图比例，优先保持竖图 `3:4` 构图。
- 背景：用户拍摄的原始餐食图片。
- 顶部：半透明深色圆角标题块，白色大字显示餐次标题。
- 中部：半透明深色信息面板，显示表格内容：
  - 名称 / 单位 / 热量 / 蛋白质
  - 每个食物一行
- 底部：黄色醒目汇总条，显示本餐热量和蛋白质。
- 底部补充区：半透明深色文本区，显示：
  - 今日累计：热量 Xkcal / 蛋白质 Xg
  - 目标：热量 Xkcal / 蛋白质 Xg
  - 今日小结：根据目标自动生成

今日小结规则：

- 热量未达标：`热量还差 Xkcal`。
- 热量达到或超过目标：`热量已达标，超出 Xkcal` 或 `热量刚好达标`。
- 蛋白质未达标：`蛋白质还差 Xg`。
- 蛋白质达到或超过目标：`蛋白质已达标，超出 Xg` 或 `蛋白质刚好达标`。

文本排版要求：

- 中文优先使用系统无衬线字体。
- 字号根据图片宽度自适应。
- 长行需要自动缩小字号或换行，不能溢出画布。
- 面板高度根据食物行数动态计算。
- 如果食物行数过多，优先缩小字号；仍放不下时提示用户拆成多餐。

## 文件命名

PNG 标注图：

```text
YYYYMMDD_餐次标题_营养标注.png
```

示例：

```text
20260701_第一餐_营养标注.png
```

MD 文件：

```text
YYYYMMDD.md
```

示例：

```text
20260701.md
```

安卓 Chrome 可能在重复下载同名文件时自动生成：

```text
20260701 (1).md
20260701 (2).md
```

这是第一版可接受行为。每次下载的 MD 都应包含当天最新完整记录。

## MD 输出格式

MD 文件包含当天完整记录：

```markdown
# 2026-07-01 饮食记录

目标：热量 2700kcal / 蛋白质 135g

## 第一餐

| 名称 | 单位 | 热量 | 蛋白质 |
| --- | --- | ---: | ---: |
| 酒酿 | 150g | 120kcal | 4g |
| 香蕉 | 1根 | 105kcal | 1g |

本餐：热量 225kcal / 蛋白质 5g

## 今日汇总

今日累计：热量 225kcal / 蛋白质 5g

今日小结：热量还差 2475kcal，蛋白质还差 130g
```

如果当天有多餐，按保存时间排序输出。

## 页面结构

第一版页面建议包含以下区域：

- 顶部目标设置区：
  - 目标热量输入框
  - 目标蛋白质输入框
- 图片输入区：
  - 拍照/选图按钮
  - 原图缩略预览
- 文本输入区：
  - 固定格式输入框
  - 格式示例提示
- 操作区：
  - 生成预览
  - 保存
  - 清空当前输入
- 预览区：
  - 标注图预览
  - 本餐汇总
  - 今日累计
- 当天记录区：
  - 已保存餐次列表
  - 删除餐次按钮

## 错误处理

- 未选择图片时，保存按钮不可用，并提示先拍照或选图。
- 输入格式错误时，显示明确错误信息，不生成文件。
- Canvas 生成失败时，提示重新选择较小图片或刷新页面。
- 浏览器下载被拦截时，提示用户允许 Chrome 下载。
- 删除餐次前需要二次确认。

## 安全和隐私

- 图片和饮食数据默认只在手机浏览器本地处理。
- 第一版不把图片上传到服务器。
- 第一版不把饮食记录上传到服务器。
- Vercel 只托管静态网页资源。
- 下载到手机后的 PNG 和 MD 文件由用户自行管理。

## 验收标准

- 安卓 Chrome 能打开 Vercel 部署页面。
- 能通过相机拍摄或相册选择图片。
- 能按固定格式解析至少 1 到 8 个食物条目。
- 能正确计算本餐热量和蛋白质。
- 能正确累计同一天多餐数据。
- 跨天后能自动创建新记录。
- 能编辑并记住热量目标和蛋白质目标。
- 能生成与现有样式相近的标注图。
- 点击保存后能下载 PNG 文件。
- 点击保存后能下载当天完整 MD 文件。
- 删除当天某餐后，今日累计和 MD 内容能同步更新。

## 测试计划

- 使用当前样例数据测试第一餐、第二餐、第三餐。
- 测试热量和蛋白质小数输入。
- 测试错误输入：
  - 缺字段
  - 热量不是数字
  - 蛋白质不是数字
  - 空标题
- 测试长食物名称和多行食物列表的图片排版。
- 测试重复保存同一天记录后的累计结果。
- 测试删除餐次后的累计结果。
- 在安卓 Chrome 真机上测试拍照、预览和下载。
- 在桌面 Chrome 上做开发调试和基础回归。

## 后续增强
